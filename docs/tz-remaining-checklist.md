# Статус реализации по ТЗ

## Обновление: блок аналитики v2

- Закрыты KPI по формуле "текущий месячный план vs прошлый факт": всего за месяц, средний чек, прогноз года и карточка потенциальной экономии только при реальных рекомендациях.
- "Динамика трат" переведена на горизонтальные бары с кликом по месяцу и bottom sheet с суммой и подписками месяца.
- Категории дополнены процентом, направлением тренда и `changePercent`, рассчитанным относительно фактических списаний прошлого месяца.
- Добавлены триалы, автоотмена триала без списания, ручные отметки использования, неиспользуемые подписки, локальные рекомендации экономии и история изменения цен.
- Детальная документация по этому блоку: [analytics-tz-implementation.md](analytics-tz-implementation.md).

## Уже реализовано

- Онбординг: 3 слайда с навигацией.
- Аутентификация: экраны ввода телефона и кода, демо-код `123456`.
- Дашборд: карточка с тратами, кошелек, блок "vs рынок", быстрые действия, список ближайших списаний, блок по категориям.
- Подписки: список с поиском, переключение "Активные / Архив", детальный экран.
- Календарь: список будущих списаний, сгруппированный по датам.
- Кошелек: баланс, кнопки пополнения, модальные окна, история транзакций.
- История списаний: переключение периода месяц / квартал / год.
- Мерчант: список созданных подписок, создание новой, копирование ссылки-заглушки, вывод, статистика.
- Профиль: отображение телефона, кнопки разделов, выход.
- Настройки: переключатели уведомлений, Face ID, отображение часового пояса.
- Роутер: настроена навигация между основными экранами.
- Локальная БД: подключен SQLite-слой для авторизации, подписок, кошелька, транзакций, мерчант-подписок и настроек.
- Архивация/удаление подписок: сценарии разделены явно. "Отменить" переносит активную подписку в архив, "Восстановить" возвращает архивную подписку в активные, "Удалить" выполняет физическое удаление записи.
- Уведомления: добавлено и проверено Android local scheduling через `flutter_local_notifications`; на эмуляторе `emulator-5554` в `dumpsys alarm` видны `RTC_WAKEUP` alarms для `com.example.subsflow` за 7/3/1 день.
- Аналитика v2: KPI, горизонтальные бары трат, категории с трендом, триалы, неиспользуемые подписки, рекомендации экономии и отдельная история изменения цен.

## Что нужно проверить или доделать

- Полнота реализации экранов: проверить, что все действия на каждом экране действительно работают, а не только отображаются.
- Уведомления Android: дополнительно проверить фактический показ уведомления через ручную смену даты/времени или ожидание ближайшего scheduled alarm.
- Уведомления iOS: проверить permission и scheduled local notifications на iOS-сборке.
- Push vs local: сейчас реализованы offline local notifications. Если по ТЗ нужны именно серверные push, отдельно потребуется backend/FCM/APNs, хранение device token и серверное расписание отправок.
- Автопополнение кошелька: проверить сценарий при старте приложения, если баланс ниже заданного порога.
- Экспорт CSV: убедиться, что экспорт содержит все подписки, включая архивные, если это требуется по ТЗ.
- Функция "Я уже оплатил": проверить, что дата списания корректно сдвигается, баланс меняется и запись появляется в истории.
- Демо-данные: проверить генерацию при входе с кодом `123456`.
- Выбор часового пояса: заменить текущую заглушку на рабочий выбор и сохранение значения.
- Редактирование подписки: реализовать действие кнопки "Редактировать" на детальном экране.
- Рекомендации экономии: после успешного подключения `url_launcher` проверить реальное открытие внешнего URL вместо fallback-сценария.
- Аналитика v2: пройти ручную QA на узком Android viewport, клики по месяцу/категории, автоотмену trial, отметку "Использовал" и `/price-history`.
- Тестирование: добавить полноценные widget/integration-тесты, проверить плавность скролла и отсутствие очевидных утечек.
- Сборка: проверить сборку под Android и iOS.
- Локализация: вынести оставшиеся hardcoded-строки в `.arb` и почистить старые битые строки, если они проявляются в UI.

## Приоритет

1. Проверить ключевые бизнес-сценарии: вход, демо-данные, оплата подписки, пополнение, история, экспорт.
2. Подтвердить фактический показ уведомлений на Android/iOS и выбор часового пояса.
3. Добавить создание / редактирование пользовательских подписок.
4. Пройтись по UI на соответствие прототипу.
5. Закрыть тесты и сборки Android / iOS.
