**Техническое задание кроссплатформенное приложение «Мои Подписки» (Flutter)**

**Прототип - https://salkalin.github.io/my-subscriptions-app-v1/**

## 1. Общие требования к коду

- Язык: **Dart 3.x**
- Фреймворк: **Flutter 3.22+**
- Совместимость: iOS 12+, Android 6+ (API 23+)
- Локализация: пока только русский (но тексты вынести в .arb)
- Зависимости: все указывать в pubspec.yaml с фиксированными версиями

## 3. Функциональные требования

### 3.1. Онбординг (3 слайда)

- Свайп или кнопка «Далее».
- Кнопка «Пропустить» → переход на экран входа.

### 3.2. Аутентификация

- Экран ввода телефона (маска + валидация).
- Чекбокс согласия с офертой.
- Экран ввода кода (демо-код 123456 — привязать к любому телефону).
- После успешного входа — генерация демо-данных (как в прототипе).
- Выход из аккаунта → очистка локальной БД.

### 3.3. Дашборд (главный экран)

- Карточка «траты в месяц» с суммой, количеством подписок, годовыми тратами, средним чеком, датой следующего списания.
- Блок кошелёк: баланс, на сколько месяцев хватит.
- Блок «vs рынок» (сравнение со средним чеком 2500 ₽).
- Быстрые действия: Добавить, Календарь, История, Экспорт.
- Список ближайших списаний (до 4).
- Блок «по категориям» с горизонтальными барами.

### 3.4. Подписки

- Список с иконками (первая буква + цвет), названием, датой до списания, ценой и периодом.
- Поиск по названию.
- Переключение «Активные / Архив».
- Нажатие на элемент → открывается детальный экран.

### 3.5. Детальный экран подписки

- Крупная иконка, название, категория, сумма, период.
- Следующее списание, статус, годовая сумма.
- Кнопки: «Я уже оплатил» (списывает с кошелька и сдвигает дату), «Редактировать», «Отменить / Удалить».

### 3.6. Календарь

- Список будущих списаний сгруппирован по датам (день + месяц).
- Индикация «сегодня / завтра / через N дней».

### 3.7. Кошелёк

- Баланс, статус автопополнения.
- Кнопки «Пополнить» и «Авто».
- Модальное окно пополнения (сумма, способ оплаты — пока заглушка).
- Модальное окно автопополнения (порог, сумма).
- Список последних транзакций (пополнения и списания).

### 3.8. История списаний

- Переключение периода: месяц / квартал / год.
- Итоговая сумма за период.
- Список операций с датами и суммами.

### 3.9. Мерчант (свои подписки)

- Список созданных пользователем подписок.
- Создание новой (название, описание, цена, период).
- Кнопка «Ссылка» — копирование ссылки-заглушки.
- Кнопка «Вывод» (минимальная сумма 100 ₽) — обнуляет earned, показывает тост.
- Статистика: количество подписчиков, заработано.

### 3.10. Профиль и настройки

- Отображение телефона.
- Разделы-кнопки: Свои подписки, Кошелёк, История, Экспорт, Настройки.
- Кнопка «Выйти».
- В настройках: переключатели уведомлений (за 7, 3, 1 день), Face ID (заглушка), выбор часового пояса.

## 4. Критерии приёмки

- Все экраны из прототипа реализованы и соответствуют дизайну.
- Все данные сохраняются после перезапуска приложения.
- Уведомления приходят в указанные сроки (имитация смены даты через ручное изменение).
- Автопополнение срабатывает при старте приложения, если баланс ниже порога.
- Экспорт CSV работает и содержит все подписки.
- Приложение собирается под iOS и Android без ошибок.
- Функция «Я уже оплатил» корректно сдвигает дату и добавляет запись в историю.
- Демо-код 123456 работает для входа с любым номером.
- Нет утечек памяти, скролл плавный.

## 5. Бонус-функции (если останется время)

- Импорт подписок из CSV / банковской выписки.
- Виджет на домашний экран (iOS и Android).

## 6. Ожидаемый результат

Работающее кроссплатформенное приложение на Flutter, которое:

- Полностью повторяет UX прототипа.
- Работает офлайн.
- Присылает push-уведомления перед списаниями.
- Позволяет управлять подписками и личным кошельком.
- Имеет чистый, поддерживаемый код с разделением на слои (presentation / domain / data).
