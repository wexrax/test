# Реализация ТЗ аналитики

Документ фиксирует текущее закрытие блока аналитики SubsFlow по ТЗ v2. Реализация остается offline-first: все расчеты строятся поверх локальной SQLite-базы, DAO/repository-слоя и Riverpod-провайдеров.

## Что закрыто

- KPI на `/analytics` пересчитаны по согласованной формуле: текущий месячный план активных подписок сравнивается с фактическими списаниями прошлого месяца.
- "Динамика трат" заменена с line/area-графика на горизонтальные бары за 6 прошлых месяцев и текущий месяц.
- Нажатие на месяц открывает расшифровку: сколько потрачено за месяц и какие подписки попали в списания.
- Категории показывают сумму, долю в текущем плане, тренд вверх/вниз/без изменений и изменение относительно фактических списаний прошлого месяца.
- Добавлены блоки триалов и неиспользуемых подписок. Пустые блоки скрываются.
- Добавлены ручные отметки использования подписки, вычисление неиспользуемости после 30 дней и скрытие такой рекомендации.
- Добавлен локальный каталог альтернатив и рекомендации экономии top-3 с учетом скрытых рекомендаций.
- Добавлен отдельный экран `/price-history` для истории изменения цен. `/history` остается историей списаний.
- Добавлены поля и таблицы SQLite v6 для триалов, использования и истории цен.
- Новые пользовательские строки вынесены в ARB и сгенерированные l10n-файлы.

## Формулы аналитики

### Всего за месяц

`Всего за месяц` считается как сумма активных подписок в месячном эквиваленте:

```text
weekly  = amount * 52 / 12
monthly = amount
yearly  = amount / 12
```

Динамика KPI сравнивает текущий месячный план с фактическими списаниями прошлого месяца.

### Средний чек

`Средний чек` считается как:

```text
текущий месячный план / количество активных подписок
```

Динамика среднего чека сравнивает текущий средний план с прошлым фактическим средним списанием.

### Прогноз года

`Прогноз года` считается как:

```text
текущий месячный план * 12
```

### Можно сэкономить

Карточка "Можно сэкономить" показывается только если есть реальные рекомендации экономии. Рекомендации строятся из локального каталога альтернатив и сортируются по годовой экономии.

## Динамика трат

Для каждого месяца берутся фактические транзакции списаний. Текущий месяц также входит в набор баров, чтобы пользователь видел текущий факт рядом с прошлыми периодами.

Данные бара включают:

- начало месяца;
- сумму списаний за месяц;
- список подписок, по которым были списания;
- признак текущего месяца.

UI показывает горизонтальные бары с относительной шириной. При нажатии открывается bottom sheet с текстом "В месяц потрачено X" и списком подписок месяца.

## Категории

Категорийная аналитика строится по активным подпискам и их месячному эквиваленту.

Для каждой категории считаются:

- текущая сумма категории;
- процент от общего текущего месячного плана;
- фактическая сумма этой категории за прошлый месяц;
- `changePercent` относительно прошлого месяца;
- направление тренда: вверх, вниз или без изменений.

Клик по категории сохраняет прежнее поведение: переход на `/subscriptions?category=...`.

## Триалы

В `subscriptions` добавлены поля:

- `trial_start_at`;
- `trial_end_at`;
- `cancel_after_trial`.

В flow добавления подписки после выбора шаблона открывается bottom sheet настроек: дата первого списания, переключатель пробного периода, дата окончания триала и переключатель "Отменить после триала".

Перед загрузкой подписок выполняется reconciliation:

- подписки с истекшим `archive_at` архивируются;
- триалы с `cancel_after_trial = 1` и истекшим `trial_end_at` архивируются без списания;
- при автоотмене триала не создается транзакция, не списывается кошелек и не сдвигается `next_payment_date`.

## Использование и неиспользуемые подписки

Модуль использования в этом этапе ручной. На детальном экране подписки добавлена кнопка "Использовал".

При нажатии:

- создается запись в `subscription_usage_events`;
- обновляется `subscriptions.last_used_at`;
- экран аналитики перестает считать подписку неиспользуемой, пока не пройдет новый 30-дневный период.

Подписка считается неиспользуемой, если она активна и `last_used_at` отсутствует или старше 30 дней. Пользователь может скрыть подсказку через `inactive_suggestion_hidden_at`.

## Рекомендации экономии

Рекомендации строятся локально, без внешнего API.

Источник данных: `lib/data/local/savings_alternative_catalog.dart`.

Алгоритм:

1. Нормализовать название подписки через aliases каталога.
2. Найти альтернативы той же категории.
3. Оставить только альтернативы дешевле текущего месячного эквивалента.
4. Посчитать месячную и годовую экономию.
5. Отфильтровать скрытые ключи из `settings.hidden_savings_recommendations`.
6. Отсортировать по годовой экономии.
7. Показать top-3.

Ограничение текущего состояния: внешний переход по URL зависит от подключения `url_launcher`. Если зависимость недоступна в окружении, кнопка должна оставаться безопасной и показывать пользователю понятный fallback.

## История изменения цен

Добавлена отдельная таблица `subscription_price_history` и экран `/price-history`.

История пополняется только если цена действительно изменилась. Запись хранит:

- `subscription_id`;
- название подписки на момент изменения;
- старую цену;
- новую цену;
- валюту;
- дату изменения.

На детальном экране подписки добавлен минимальный dialog изменения цены, чтобы историю можно было наполнить из UI.

## Локализация

Новые строки аналитики, триалов, использования, рекомендаций и истории цен добавляются в:

- `lib/l10n/app_ru.arb`;
- `lib/l10n/app_en.arb`;
- сгенерированные `app_localizations*.dart`.

В старых экранах еще остаются hardcoded-строки, но новый пользовательский текст этого блока должен проходить через `AppLocalizations`.

## Основные файлы

- `lib/data/local/app_database.dart` - миграция схемы v6.
- `lib/data/local/subscription_dao.dart` - trial reconciliation, usage events, price history.
- `lib/data/local/savings_alternative_catalog.dart` - локальный каталог альтернатив.
- `lib/data/models/subscription_model.dart` - новые поля подписки.
- `lib/data/models/subscription_usage_event_model.dart` - модель usage event.
- `lib/data/models/subscription_price_history_model.dart` - модель истории цен.
- `lib/presentation/providers/subscription_provider.dart` - публичные методы подписок и price history provider.
- `lib/presentation/providers/analytics_provider.dart` - KPI, бары, категории, триалы, inactive, рекомендации.
- `lib/presentation/screens/analytics/analytics_screen.dart` - новый UI аналитики.
- `lib/presentation/screens/add_subscription/add_subscription_screen.dart` - настройки триала при добавлении.
- `lib/presentation/screens/subscriptions/subscription_detail_screen.dart` - отметка использования и изменение цены.
- `lib/presentation/screens/history/price_history_screen.dart` - история изменения цен.
- `lib/presentation/router/app_router.dart` - route `/price-history`.
- `test/backend_data_test.dart` - тесты миграции, trial auto-cancel, usage, KPI, рекомендации и price history.
